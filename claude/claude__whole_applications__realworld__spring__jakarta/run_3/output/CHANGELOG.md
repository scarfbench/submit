## [2026-03-13T14:32:00Z] [error] SQL Schema Compatibility Issue
- **File**: src/main/resources/schema.sql
- **Error**: H2 2.2.224 (bundled with Spring Boot 3.2.5) rejected `BIGSERIAL` and `AUTO_INCREMENT`
  in PostgreSQL compatibility mode, and `value` is now a reserved keyword
- **Root Cause**: H2 upgraded from 1.x to 2.x between Spring Boot 2.x and 3.x with stricter SQL parsing
- **Resolution**:
  - Replaced `BIGSERIAL PRIMARY KEY` with `BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY`
  - Replaced `AUTO_INCREMENT` with `GENERATED BY DEFAULT AS IDENTITY`
  - Quoted reserved keyword `value` as `"value"` in tags table
  - Updated Tag.java `@Column` annotation to use escaped quotes: `name = "\"value\""`

## [2026-03-13T14:35:00Z] [error] JWT Authentication Failure (Spring Security 6)
- **Error**: All authenticated endpoints returning 401 Unauthorized
- **Root Cause**: Spring Security 6 changed `SecurityContextHolder` behavior; setting authentication
  directly in `SecurityContextHolder.getContext()` from a filter no longer persists properly
- **Resolution**: Refactored `JWTAuthenticationFilter` to use `AuthenticationManager.authenticate()`
  which properly delegates to `JWTAuthenticationProvider` and creates an authenticated token
